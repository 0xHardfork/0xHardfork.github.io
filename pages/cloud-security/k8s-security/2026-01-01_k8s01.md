# Kubernetes Security

## 1. Basic Resource Viewing

### Node Management
```bash
# View the status of all nodes, showing details like IP, uptime, version, OS, etc.
kubectl get nodes -o wide

# View detailed information of a single node (Events, resource allocation, status)
kubectl describe node <node-name>

# Real-time monitoring of node resource consumption (if metrics-server is installed)
kubectl top nodes
```

### Workload & Container (Pod & Deployment) Management
```bash
# View all Pods (across all Namespaces)
kubectl get pods -A

# View the detailed topology of Pods in the default Namespace (or specify -n Namespace)
kubectl get pods -o wide -n default

# View all deployments, services, and pods
kubectl get deployments,services,pods 

# View detailed scheduling information and error logs of a Pod (*great for troubleshooting*)
kubectl describe pod <pod-name> -n <namespace>
```

## 2. Daily Deployment & Interaction

### Quickly Create and Expose Services
```bash
# Create a test Deployment (e.g., Nginx, 3 replicas)
kubectl create deployment nginx-test --image=nginx --replicas=3

# Expose this Deployment (mapped as NodePort for external access via NodeIP:Port)
kubectl expose deployment nginx-test --port=80 --type=NodePort

# Or just quickly spin up a one-off Pod for network environment testing
kubectl run curl-test --image=curlimages/curl --rm -it --restart=Never -- sh
```

### Entering and Viewing Containers
```bash
# Real-time track/tail the output logs of a Pod
kubectl logs -f <pod-name> -n <namespace>

# (If the Pod has multiple containers, you need to specify the exact container name)
kubectl logs -f <pod-name> -c <container-name>

# Enter inside a Pod to execute interactive commands
kubectl exec -it <pod-name> -- /bin/bash
# (Sometimes you can't enter bash, try sh)
kubectl exec -it <pod-name> -- /bin/sh
```

## 3. Modifying and Deleting Resources

```bash
# Edit existing resource objects (opens vi/nano to modify YAML)
kubectl edit deployment/<deployment-name> -n <namespace>

# Scale the number of replicas
kubectl scale deployment nginx-test --replicas=5

# Rollback the latest update/release
kubectl rollout undo deployment/nginx-test

# Delete a specific Pod (it will be automatically restarted by its ReplicaSet)
kubectl delete pod <pod-name>

# Completely clean up all associated resources (by label or name)
kubectl delete deployment nginx-test
kubectl delete service nginx-test
```

## 4. Node Maintenance & Troubleshooting

### Node Maintenance Mode (Drain)
When your Worker node needs to be restarted or maintenance/upgrade the system:
```bash
# Prevent new Pods from being scheduled to the node, and evict existing Pods on the node (moving them to other nodes)
kubectl drain worker-1 --ignore-daemonsets --delete-emptydir-data

# Maintenance is finished, uncordon the node
kubectl uncordon worker-1
```

### Common System-Level Troubleshooting (Requires SSH login to the target node)
```bash
# 1. Check if the kubelet status and logs can communicate with apiserver
systemctl status kubelet
journalctl -u kubelet -f

# 2. Check the status of containerd (the underlying container runtime)
systemctl status containerd
journalctl -u containerd -f

# 3. Check the system CNI interface or zombie networks
ip a
```

## 5. Cilium Network & Security Exclusive Operations

Because you use `Cilium` as the network plugin and have `Hubble` enabled, you get extremely powerful network observability:

```bash
# View the status of Hubble Relay and Cilium components
kubectl get pods -n kube-system -l k8s-app=cilium

# Temporarily forward Hubble UI panel traffic to the host machine's browser (Execute on Master)
kubectl port-forward -n kube-system svc/hubble-ui --address 0.0.0.0 8888:80
```
