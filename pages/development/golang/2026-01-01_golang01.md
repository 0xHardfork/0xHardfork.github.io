# Golang

## 🧠 Concept Overview

Go (or Golang) is a statically typed, compiled language designed at Google by Robert Griesemer, Rob Pike, and Ken Thompson. It was built to solve real engineering problems: slow build times, complex dependency management, and the difficulty of writing concurrent software. Go is famous for its simplicity — the entire language spec can be read in a few hours — yet it scales to power Docker, Kubernetes, and much of modern cloud infrastructure.

At its core, Go organizes code into **packages**. Every Go file begins with `package <name>`, which declares which package the file belongs to. The special package name `main` signals to the Go toolchain that this is an executable program, not a library. The `main()` function inside `main` package is the entry point — execution always starts there.

Go's toolchain ships as a single binary (`go`) with everything you need: compiler, linter, test runner, formatter, and module manager. This "batteries included" philosophy means you don't have to assemble a toolchain — you just install Go and start writing.

---

## 🔑 Key Points

- Every Go source file starts with `package <name>`; executable programs use `package main`
- `func main()` is the required entry point for any runnable Go program
- `import` loads other packages; standard library packages use short names like `"fmt"`
- Go code is formatted with `gofmt` (or `go fmt ./...`) — there is one canonical style, no debates
- `go run <file>.go` compiles and runs in one step; `go build` produces a binary
- The `fmt` package handles all printing: `fmt.Println`, `fmt.Printf`, `fmt.Sprintf`
- Go module system (`go mod`) manages dependencies — every project needs a `go.mod` file

## 💻 Code Examples

### Example 1 — Hello, Go!

```go
// File: hello.go
package main

import "fmt"

func main() {
	// fmt.Println prints the arguments followed by a newline
	fmt.Println("Hello, Go!")

	// fmt.Printf uses format verbs — %s for strings, %d for integers, %v for anything
	name := "Gopher"
	age := 14
	fmt.Printf("Name: %s, Age: %d\n", name, age)

	// fmt.Sprintf returns a formatted string (doesn't print)
	greeting := fmt.Sprintf("Welcome, %s!", name)
	fmt.Println(greeting)
}
```

**What this demonstrates**: The minimal structure of any Go program — `package main`, `import`, and `func main()`. Also shows the three most common `fmt` functions and the `:=` short variable declaration syntax.

---

### Example 2 — A More Complete First Program

```go
// File: about_me.go
package main

import (
	"fmt"
	"runtime"
)

func main() {
	// Go lets you import multiple packages in a single grouped import block
	// runtime gives us info about the Go runtime environment

	// Declare variables using different styles
	var language string = "Go"       // explicit type + value
	version := runtime.Version()     // short declaration, type inferred
	const maxGoroutines = 1_000_000  // constants use 'const' keyword

	fmt.Println("=== About My Setup ===")
	fmt.Printf("Language  : %s\n", language)
	fmt.Printf("Version   : %s\n", version)
	fmt.Printf("OS        : %s\n", runtime.GOOS)
	fmt.Printf("Arch      : %s\n", runtime.GOARCH)
	fmt.Printf("Max goroutines possible: %d\n", maxGoroutines)
}
```

**What this demonstrates**: Grouped imports, the three ways to declare variables in Go (`var`, `:=`, `const`), and using a second standard library package (`runtime`) to pull environment info at runtime.

## ⚠️ Common Mistakes

| Mistake | Why it's wrong | Correct approach |
|---------|---------------|-----------------|
| `import "fmt"` then never using it | Go **won't compile** if you import a package you don't use | Remove unused imports; your editor (with gopls) will auto-manage these |
| Declaring a variable but never using it | Same rule — unused local variables are a compile error | Use `_` (blank identifier) if you intentionally discard a value |
| Writing `func Main()` (capital M) | Go is case-sensitive; the entry point must be lowercase `main` | Always `func main()` |
| Forgetting `package main` at the top | The compiler won't know this is an executable | Every `.go` file needs a `package` declaration as its first non-comment line |
| Running `go run .` in a directory with no `go.mod` | Works for single files, but multi-file projects need a module | Run `go mod init <module-name>` once per project |

## 🏋️ Practice Exercises

### Exercise 1 — Personal Info Printer (Easy)

**Goal**: Write a Go program that prints your name, your favorite programming language, and the current Go version.

**Requirements**:
- Use `package main` and `func main()`
- Declare at least one variable with `var` and at least one with `:=`
- Use both `fmt.Println` and `fmt.Printf`
- Use `runtime.Version()` to print the Go version

**Starter code**:

```go
package main

import (
	"fmt"
	"runtime"
)

func main() {
	// TODO: Declare your name using 'var'
	// TODO: Declare your favorite language using ':='
	// TODO: Print all three pieces of info (name, favorite language, Go version)
}
```

**Hint**: `runtime.Version()` returns a string like `"go1.22.0"` — you can use it directly in `fmt.Printf`.

### Exercise 2 — Simple Calculator (Medium)

**Goal**: Build a tiny command-line calculator that prints the result of some hardcoded arithmetic operations.

**Requirements**:
- Declare `a` and `b` as integer constants (e.g., `42` and `7`)
- Compute and print: sum, difference, product, quotient, and remainder
- Use a `const` block for declaring `a` and `b`
- Format output neatly with aligned columns using `fmt.Printf`

**Starter code**:

```go
package main

import "fmt"

func main() {
	const (
		a = 42
		b = 7
	)

	// TODO: Compute sum, difference, product, quotient, remainder
	// TODO: Print them in a formatted table like:
	//   42 + 7  = 49
	//   42 - 7  = 35
	//   ...
}
```

**Hint**: The remainder (modulo) operator in Go is `%`. In `fmt.Printf`, use `%-8s` to left-align a string in a field of width 8.

### Exercise 3 — System Info Banner (Hard / Challenge)

**Goal**: Write a program that prints a nicely formatted system information banner, combining what you know about `fmt`, `runtime`, and string formatting.

**Requirements**:
- Print a top and bottom border made of `=` characters (exactly 40 wide)
- Print the following fields centered or left-aligned inside the banner:
  - Go Version
  - Operating System (`runtime.GOOS`)
  - CPU Architecture (`runtime.GOARCH`)
  - Number of CPUs (`runtime.NumCPU()`)
- Use `fmt.Sprintf` to build each line, then `fmt.Println` to print it
- All fields must be aligned — use format width specifiers in `fmt.Printf`

**Starter code**:

```go
package main

import (
	"fmt"
	"runtime"
	"strings"
)

func main() {
	border := strings.Repeat("=", 40)

	// TODO: Print the banner with the border at top and bottom
	// TODO: Print each field neatly between the borders
	// Example output:
	// ========================================
	//   Go Version : go1.22.0
	//   OS         : linux
	//   Arch       : amd64
	//   CPUs       : 8
	// ========================================

	_ = border // remove this once you use border
	_ = runtime.Version // remove this once you use runtime
}
```

**Hint**: `fmt.Printf("  %-12s: %s\n", "Go Version", runtime.Version())` will give you left-aligned labels with consistent spacing. `runtime.NumCPU()` returns an `int` — use `%d` for it.

## 🗒️ Quick Cheat Sheet

```go
// Minimal runnable Go program
package main

import "fmt"

func main() {
    var x string = "explicit"  // var declaration
    y := 42                    // short declaration (type inferred)
    const z = 3.14             // constant

    fmt.Println(x)             // print with newline
    fmt.Printf("%d\n", y)      // formatted print
    s := fmt.Sprintf("%.2f", z) // format to string
    fmt.Println(s)

    // go run hello.go     → compile & run
    // go build hello.go   → produce binary
    // go fmt ./...        → format all files
    // go mod init myapp   → initialize a module
}
```
