# Golang

## 🧠 Concept Overview

Go's standard library is famously robust, often eliminating the need for third-party frameworks for HTTP servers, JSON parsing, or crypto. 

At the core of Go's design are small, composable interfaces. The two most omnipresent are **`io.Reader`** and **`io.Writer`**. Unlike object-oriented inheritance where you might extend an `InputStream`, in Go, anything that implements a `Read(p []byte) (n int, err error)` method *is* an `io.Reader`. This allows you to compose an HTTP request body, a file, an in-memory buffer, or a compressed tarball through the exact same pipeline without them knowing about each other.

Another critical pillar of Go system design is the **`context.Context`**. Go doesn't have thread-local storage or an overarching exception mechanism. If you fire off a goroutine to make an HTTP request, and the user cancels the operation, you need a way to propagate that cancellation down the call stack to abort the network dial. `context.Context` is passed explicitly as the *first argument* to almost every blocking or long-running function in Go to handle deadlines, cancellation signals, and request-scoped values.

---

## 🔑 Key Points

- **`io.Reader` and `io.Writer` are everywhere**: Files, network connections, buffers, and hashers all use these interfaces.
- **Composition over Inheritance**: Use wrappers like `bufio.NewReader`, `gzip.NewWriter`, or `io.MultiReader` to chain behaviors together without creating monolithic classes.
- **Zero-copy where possible**: Use `io.Copy(dst, src)` to move data efficiently without loading everything into memory.
- **`context.Context` propagates cancellation**: It forms a tree. Canceling a parent context immediately cancels all its children.
- **Context is immutable**: You don't mutate a context; you derive a new one (e.g., `context.WithTimeout(parent, time.Second)`).
- **JSON encoding works with streams**: `json.NewDecoder(io.Reader)` is vastly more memory-efficient for large payloads than `json.Unmarshal(byte[])`.

## 💻 Code Examples

### Example 1 — Composable I/O pipelines

```go
// File: example1.go
package main

import (
	"bytes"
	"compress/gzip"
	"fmt"
	"io"
	"os"
	"strings"
)

func main() {
	// A strings.Reader implements io.Reader
	source := strings.NewReader("Hello, composable world of Go I/O!")
	
	// An in-memory buffer implements both io.Reader and io.Writer
	var compressed bytes.Buffer
	
	// gzip.Writer wraps an io.Writer (our buffer)
	gzWriter := gzip.NewWriter(&compressed)
	
	// io.Copy streams from an io.Reader to an io.Writer.
	// We are streaming a string, compressing it, and writing to memory.
	// This happens efficiently using a small internal buffer.
	io.Copy(gzWriter, source)
	gzWriter.Close() // Flush the gzip trailer
	
	fmt.Printf("Compressed size: %d bytes\n", compressed.Len())
	
	// Now, let's read it back directly into Standard Out
	gzReader, _ := gzip.NewReader(&compressed)
	
	// os.Stdout is just an *os.File, which implements io.Writer
	fmt.Print("Decompressed: ")
	io.Copy(os.Stdout, gzReader)
	fmt.Println()
}
```

**What this demonstrates**: How Go's interface-driven design allows chaining readers and writers (`strings.Reader` -> `gzip.Writer` -> `bytes.Buffer` -> `gzip.Reader` -> `os.Stdout`) seamlessly using `io.Copy` without intermediate `[]byte` allocations.

### Example 2 — Context Timeout and HTTP Client

```go
// File: example2.go
package main

import (
	"context"
	"fmt"
	"io"
	"net/http"
	"time"
)

func fetchDataview(ctx context.Context, url string) error {
	// Derive a new context that cancels after 500ms
	fetchCtx, cancel := context.WithTimeout(ctx, 500*time.Millisecond)
	defer cancel() // MUST ALWAYS DEFER CANCEL to prevent resource leaks

	// Create request with the derived context
	req, err := http.NewRequestWithContext(fetchCtx, "GET", url, nil)
	if err != nil {
		return err
	}

	// Make the HTTP call. If 500ms elapse, the HTTP client will abort the TCP connection
	// and return a context.DeadlineExceeded error.
	resp, err := http.DefaultClient.Do(req)
	if err != nil {
		return fmt.Errorf("request failed: %w", err)
	}
	defer resp.Body.Close()

	if resp.StatusCode != http.StatusOK {
		return fmt.Errorf("bad status: %d", resp.StatusCode)
	}
	
	// Read a maximum of 100 bytes using io.LimitReader to avoid memory exhaustion
	body, _ := io.ReadAll(io.LimitReader(resp.Body, 100))
	fmt.Printf("Received: %s...\n", body)
	
	return nil
}

func main() {
	// The root context, never cancelled
	ctx := context.Background()
	
	// Hitting an API that simulates a 2-second delay
	err := fetchDataview(ctx, "https://httpstat.us/200?sleep=2000")
	if err != nil {
		// Output: request failed: Get "...": context deadline exceeded
		fmt.Println("Error:", err)
	}
}
```

**What this demonstrates**: How to properly create and propagate timeouts across network boundaries using `context.WithTimeout` and `http.NewRequestWithContext`, preventing hanging network calls and goroutine leaks.

## ⚠️ Common Mistakes

| Mistake | Why it's wrong | Correct approach |
|---------|---------------|-----------------|
| Forgetting `defer cancel()` | The resources allocated by the Context timer (like a background goroutine) will leak until the timer naturally expires, even if the primary operation finishes early. | Always pair `ctx, cancel := context.With...` with `defer cancel()`. |
| Passing contexts inside structs | Context is execution-scoped (request-scoped), not object-scoped. Storing it in a struct blurs lifetimes and causes bugs. | Pass `ctx context.Context` as the explicit first parameter of methods. |
| Using `json.Unmarshal` for huge payloads | It loads the entire JSON document into a `[]byte` slice in memory, causing OOM (Out Of Memory) crashes. | Use `json.NewDecoder(resp.Body).Decode(&v)` to decode directly from the network stream in chunks. |
