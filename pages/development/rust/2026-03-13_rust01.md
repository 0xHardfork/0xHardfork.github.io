# Rust

## 🧠 Concept Overview

As an experienced engineer, you already know what variables, data types, and functions are. In Rust, the focus shifts to **compiler guarantees** and **memory semantics**. Rust is fundamentally an **expression-oriented** language with strict static typing. 

Here is what makes Rust's fundamentals different from languages like C++, Java, or Go:
1. **Immutable by Default**: When you bind a variable using `let`, neither the memory it points to nor the value itself can be changed. You must explicitly opt-in to mutability using `let mut`. This eliminates a huge class of concurrency and state bugs at compile time.
2. **Shadowing vs. Reassignment**: You can use `let` again with the same name. This is called *shadowing*, which effectively destroys the old binding and creates a brand-new one. Unlike reassignment, shadowing allows you to change the type of the variable or safely transform immutable data without declaring `mut`.
3. **Everything is an Expression**: Almost all constructs in Rust evaluate to a value. This means `if`, `match`, and block scopes `{}` can be directly assigned to variables. Statements (which don't return a value, like `let x = 5;`) end with a semicolon `;`, while expressions do not. Returning a value from a block is done simply by omitting the semicolon on the final line.

---

## 🔑 Key Points

- **`let` vs `let mut`**: Deterministic control over where state changes are allowed.
- **Statements vs Expressions**: Statements do an action and evaluate to `()` (the unit type). Expressions evaluate to a concrete value. *Trailing semicolons turn expressions into statements.*
- **No Implicit Type Conversion**: Rust will not silently coerce an `i32` to an `i64`. Conversions must be explicit using `as` or the `From`/`Into` traits.
- **No `null` or `nil`**: Rust handles the absence of values at the type-system level using `Option<T>`.

## 💻 Code Examples

### Example 1 — Mutability vs. Shadowing

```rust
// File: main.rs
fn main() {
    // --- 1. Mutability ---
    let mut mutable_counter = 0;
    mutable_counter = 1; // Pure reassignment (same type and memory location)

    // --- 2. Shadowing ---
    let process: i32 = 1234;
    // We create a new variable that shadows the previous `process`.
    // It can even be a completely different type.
    let process: String = process.to_string(); 
    
    println!("Counter: {}, Process ID: {}", mutable_counter, process);
}
```

**What this demonstrates**: Mutating a variable requires `mut` and preserves its type, while shadowing (`let` again) creates a brand new binding, allowing type transformation without generating throwaway variable names like `process_str`.

### Example 2 — Expression-Oriented Flow

```rust
// File: main.rs
fn process_code(code: i32) -> &'static str {
    // `if` is an expression. We assign its result directly to `status`.
    // Note: No semicolons at the end of "Success" or "Error" branches.
    let status = if code == 200 {
        "Success"
    } else {
        "Error"
    };

    // A block `{}` is an expression. The last line without a semicolon
    // implicitly returns the value representing the function's return.
    status
}

fn main() {
    let result = process_code(200);
    println!("Status: {}", result);
}
```

**What this demonstrates**: Getting rid of intermediate mutable variables. Instead of declaring `let mut status = "";` and returning it, we directly capture the value of the `if` block, preventing uninitialized state issues.

## ⚠️ Common Mistakes

| Mistake | Why it's wrong | Correct approach |
|---------|---------------|-----------------|
| `let x = 5; x = 6;` | Variables are immutable by default. The compiler rejects reassignment. | Use `let mut x = 5;` |
| `if status { 1; } else { 0; }` | Adding `;` to the block's end turns it into a statement returning `()`, violating type expectations. | Remove the trailing semicolons: `if status { 1 } else { 0 }` |
| `let n: i64 = 5i32;` | Rust has no implicit numeric casts to prevent silent precision loss bugs. | Explicit cast: `let n: i64 = 5i32 as i64;` |

## 🏋️ Practice Exercises

### Exercise 1 — Expression Refactoring (Easy)

**Goal**: Refactor the traditional imperative code snippet to a more idiomatic rust expression.

**Requirements**:
- Remove the `let mut output;` binding.
- Directly assign the result of the `if` statement to `output`.

**Starter code**:

```rust
fn main() {
    let mode = "admin";
    
    let mut output = 0;
    if mode == "admin" {
        output = 1;
    } else {
        output = 0;
    }
    
    println!("Output index: {}", output);
}
```

**Hint**: Let your `if` block return the number directly, and assign that to `let output = ...;` (don't forget the semicolon at the end of the assignment!).

## 🗒️ Quick Cheat Sheet

```rust
// Mutability
let a = 1;      // immutable
let mut b = 2;  // mutable
let a = "str";  // shadowed (type changed!)

// Block expression
let val = {
    let temp = b * 2;
    temp + a.len() // No semicolon -> returned to val
};
```
